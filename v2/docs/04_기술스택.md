# PDFS 기술 스택

> v2 서버형 구조 기준 기술 스택 정리  
> 최종 수정: 2026-06-14

---

## 1. 기술 선정 기준

HPDFS는 사용자의 로컬 PC에서 SMART 정보를 수집해야 하므로, 일반 웹 브라우저만으로는 구현할 수 없다. 따라서 로컬 수집 Agent와 서버형 대시보드를 분리한 구조를 사용한다.

---

## 2. 전체 기술 스택

| 영역 | 기술 | 역할 |
|------|------|------|
| Agent | Python, smartctl | 로컬 PC의 HDD/SSD SMART 정보 수집 |
| Backend | FastAPI | 진단 API, 모델 예측, DB 저장 |
| ML | scikit-learn, joblib | RandomForest 모델 로딩 및 예측 |
| DB | PostgreSQL | 디스크 최신 상태 및 진단 이력 저장 |
| Frontend | Streamlit | 모니터링 대시보드 제공 |
| Proxy | Nginx | `/api/*`와 대시보드 요청 분리 |
| Infra | Docker Compose | 서비스 통합 실행 |
| 확장 배포 | Kubernetes | Pod 관리 및 배포 구조 설계 |
| CI/CD | GitHub Actions | 이미지 빌드 및 배포 자동화 구조 |

---

## 3. 머신러닝 스택

| 항목 | 내용 |
|------|------|
| 라이브러리 | scikit-learn |
| 최종 모델 | RandomForestClassifier |
| 모델 저장 | joblib `.pkl` |
| 모델 파일 | `v2/backend/models/storage_best_model.pkl` |
| Threshold 파일 | `v2/backend/models/storage_threshold.json` |
| Threshold | 0.36 |

RandomForest 모델은 SMART 지표 기반 고장 확률을 계산한다. Backend는 모델 확률과 SMART 규칙 기반 판단을 결합하여 최종 위험 등급을 산출한다.

---

## 4. Backend 구조

```text
v2/backend/
├── main.py
├── database.py
├── models.py
├── predictor.py
├── router_diagnose.py
├── router_disks.py
├── models/
│   ├── storage_best_model.pkl
│   └── storage_threshold.json
└── tests/
```

| 파일 | 역할 |
|------|------|
| `main.py` | FastAPI 앱 생성 및 라우터 등록 |
| `database.py` | DB 연결 및 Session 관리 |
| `models.py` | SQLAlchemy 테이블 모델 정의 |
| `predictor.py` | ML 예측, threshold 적용, SMART 규칙 보정 |
| `router_diagnose.py` | 진단 요청 API |
| `router_disks.py` | 디스크 목록/상세 조회 API |

---

## 5. Frontend 구조

```text
v2/frontend/
├── app.py
├── requirements.txt
└── Dockerfile
```

Streamlit 대시보드는 Backend API를 호출하여 디스크 상태와 진단 이력을 표시한다.

---

## 6. Docker Compose 구성

```text
v2/docker-compose.yml
  ├── postgresql
  ├── backend
  ├── frontend
  └── nginx
```

Docker Compose를 통해 Backend, Frontend, PostgreSQL, Nginx를 한 번에 실행할 수 있다.

---

## 7. 테스트

Backend 테스트는 pytest로 구성되어 있으며, predictor 로직과 diagnose API를 검증한다.

```bash
cd v2/backend
set DATABASE_URL=sqlite:///./test.db
py -m pytest tests/ -v
```

현재 기준 테스트 결과는 22개 통과이다.
