# HPDFS 핵심 기능 정의

> SMART 기반 저장장치 고장 예측 및 모니터링 시스템  
> 최종 수정: 2026-06-14 · v2 기준

---

## 1. MVP 범위

HPDFS v2의 MVP는 **저장장치 SMART 데이터 수집 → 고장 위험 예측 → DB 저장 → 대시보드 확인** 흐름을 완성하는 것이다.

배터리 예측, 증상 기반 부품 진단, RAM 진단은 현재 범위에서 제외한다.

## 3. 기능별 상세

### F-01. 저장장치 고장 예측 모델 학습

- 데이터: Backblaze Hard Drive Test Data
- 최종 데이터 구성: 정상 45,500개, 고장 5,000개
- 모델: RandomForestClassifier
- 입력 피처: `model`, `capacity_bytes`, `smart_5_raw`, `smart_9_raw`, `smart_187_raw`, `smart_188_raw`, `smart_194_raw`, `smart_197_raw`, `smart_198_raw`, `smart_199_raw`
- 모델 파일: `v2/backend/models/storage_best_model.pkl`

### F-02. Threshold 최적화

- 기본 0.5 기준이 아니라, 고장 클래스 F2-score 기준으로 threshold를 탐색하였다.
- 최종 threshold는 `0.36`이다.
- threshold 정보는 `v2/backend/models/storage_threshold.json`에서 로드한다.

### F-03. SMART 수집 Agent

- `smartctl --scan`으로 디스크 목록을 탐색한다.
- `smartctl -a -j`로 SMART 정보를 JSON 형태로 수집한다.
- 필요한 SMART raw 값을 추출해 Backend `/api/diagnose`로 전송한다.

### F-04. 진단 API

- FastAPI Backend가 Agent 또는 수동 입력 데이터를 수신한다.
- RandomForest 모델이 고장 확률을 계산한다.
- SMART 규칙 기반 등급과 결합하여 최종 등급을 반환한다.

### F-05. 규칙 기반 위험도 보정

다음 SMART 지표는 ML 예측 확률이 낮아도 위험도를 상향하는 데 사용한다.

| SMART | 의미 | 위험 판단 |
|------|------|-----------|
| 5 | 재할당 섹터 수 | 100 이상 위험, 1 이상 주의 |
| 187 | 정정 불가 에러 보고 | 1 이상 위험 |
| 188 | 명령 타임아웃 | 1 이상 주의 |
| 194 | 온도 | 50℃ 이상 주의 |
| 197 | 대기 섹터 수 | 1 이상 위험 |
| 198 | 오프라인 정정 불가 섹터 | 1 이상 위험 |
| 199 | UDMA CRC 오류 | 1 이상 주의 |

### F-06. DB 저장

- `disks`: 디스크별 최신 상태 저장
- `diagnosis_logs`: 진단 이력 저장

### F-07. 대시보드

- 전체 디스크 수, 정상/주의/위험 개수 표시
- 디스크 목록 표시
- 디스크 상세 정보 표시
- 진단 이력 조회

---

## 4. 기능 흐름

```text
Agent 실행
  → smartctl로 SMART 수집
  → Backend /api/diagnose 전송
  → RandomForest 모델 예측
  → SMART 규칙 기반 보정
  → PostgreSQL 저장
  → Streamlit Dashboard 조회
```

---

## 5. 목표 눈높이

HPDFS의 목표는 완벽한 고장 예측이 아니라 **고장 가능성이 높은 저장장치를 사전에 경고하는 것**이다. 따라서 모델 선정에서는 Accuracy보다 고장 클래스 Recall과 F2-score를 중요하게 본다.
