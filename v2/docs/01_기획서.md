# PDFS 프로젝트 기획서

> Predictive Drive Failure System  
> SMART 기반 저장장치 고장 예측 및 모니터링 시스템  
> 최종 수정: 2026-06-14 · 상태: v2 구현 및 모델 갱신 완료

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | HPDFS (Heopung Predictive Drive Failure System) |
| 주제 | SMART 기반 저장장치 고장 위험 예측 및 모니터링 |
| 한 줄 정의 | HDD/SSD의 SMART 데이터를 수집하여 고장 위험도를 예측하고, 대시보드에서 상태를 확인하는 시스템 |
| 핵심 대상 | HDD/SSD 저장장치 |
| 프로젝트 형태 | 로컬 Agent + FastAPI Backend + PostgreSQL + Streamlit Dashboard |

HPDFS는 사용자의 PC에 장착된 HDD/SSD의 SMART 정보를 수집하고, 학습된 머신러닝 모델과 규칙 기반 판단을 함께 사용하여 저장장치의 고장 위험도를 정상/주의/위험 등급으로 제공하는 시스템이다.

---

## 2. 배경 및 목적

일반 사용자는 재할당 섹터, 대기 섹터, 정정 불가 섹터, CRC 오류와 같은 SMART 수치를 직접 해석하기 어렵다. 저장장치 고장은 갑자기 발생하는 것처럼 보이지만, 실제로는 SMART 지표에 이상 징후가 먼저 나타나는 경우가 있다.

HPDFS의 목적은 다음과 같다.

| 목적 | 설명 |
|------|------|
| 고장 위험 사전 감지 | 고장 가능성이 높은 저장장치를 미리 탐지 |
| 사용자 이해도 향상 | 복잡한 SMART 값을 정상/주의/위험 등급으로 단순화 |
| 대응 조치 제공 | 백업, 케이블 점검, 교체 검토 등 권장 조치 안내 |
| 중앙 모니터링 확장 | 여러 PC의 디스크 상태를 서버에서 관리할 수 있는 구조 제공 |

---

## 3. 프로젝트 범위

| 대상 | 사용 여부 | 설명 |
|------|:---:|------|
| HDD/SSD 저장장치 | ✅ | Backblaze HDD SMART 데이터 기반 모델 학습 및 실제 진단 구현 |
| 배터리 | ❌ | 현재 v2 범위에서 제외 |
| 증상 기반 부품 진단 | ❌ | 현재 v2 범위에서 제외 |
| RAM | ❌ | 공개 데이터 및 수집 난이도 문제로 제외 |

현재 최종 구현 범위는 **저장장치 고장 예측**이다. 배터리와 증상 기반 진단은 초기 검토 대상이었으나, 최종 발표 및 v2 구조에서는 제외하였다.

## 4. 시스템 구조

### v2 서버형 구조

```text
[사용자 PC]
  Agent
  └─ smartctl로 HDD/SSD SMART 정보 수집
        ↓ HTTP POST

[Server]
  Nginx
  ├─ /api/* → FastAPI Backend
  │             ├─ RandomForest 모델 예측
  │             ├─ SMART 규칙 기반 위험도 보정
  │             └─ PostgreSQL 저장
  └─ /     → Streamlit Dashboard
```

## 5. 데이터셋

| 항목 | 내용 |
|------|------|
| 출처 | Backblaze Hard Drive Test Data |
| 초기 데이터 | 2016년 1분기 데이터 기반 |
| 개선 방식 | 정상 데이터는 유지하고, 2016년 2분기 이후 데이터를 추가 활용하여 고장 사례 확장 |
| 최종 데이터 수 | 50,500개 |
| 정상 데이터 | 45,500개 |
| 고장 데이터 | 5,000개 |
| 고장 비율 | 약 9.9% |

초기에는 2016년 1분기 데이터만 사용하여 고장 사례가 336개로 적었다. 이후 정상 데이터 수는 유지하고 2016년 2분기 이후 데이터를 추가 수집하여 고장 사례를 5,000개까지 확장하였다.

## 6. 최종 모델

| 항목 | 내용 |
|------|------|
| 최종 모델 | RandomForestClassifier |
| 선택 기준 | 고장 클래스 F2-score 중심 |
| Threshold | 0.36 |
| 입력 피처 | model, capacity_bytes, SMART 5/9/187/188/194/197/198/199 raw 값 |
| 모델 파일 | `v2/backend/models/storage_best_model.pkl` |
| Threshold 파일 | `v2/backend/models/storage_threshold.json` |

고장 예측 문제에서는 실제 고장을 놓치지 않는 것이 중요하므로, 단순 Accuracy보다 고장 클래스 Recall과 F2-score를 중심으로 모델을 선정하였다.

## 8. 기대 효과

- 사용자가 SMART 수치를 직접 해석하지 않아도 저장장치 위험도를 확인할 수 있다.
- 고장 위험이 높은 저장장치에 대해 사전 백업 및 교체 판단을 도울 수 있다.
- Agent-Backend-DB-Dashboard 구조를 통해 다중 PC 중앙 모니터링 구조로 확장할 수 있다.
