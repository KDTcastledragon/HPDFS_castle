# PDFS 화면 설계

> Streamlit 기반 저장장치 모니터링 대시보드 화면 설계  
> 최종 수정: 2026-06-14

---

## 1. 화면 설계 원칙

HPDFS는 저장장치 상태를 모니터링하는 관제형 시스템이다. 따라서 사용자가 전체 상태를 빠르게 파악하고, 위험 디스크를 선택해 상세 원인을 확인할 수 있도록 구성한다.

| 원칙 | 적용 |
|------|------|
| Overview First | 대시보드 첫 화면에서 정상/주의/위험 개수 표시 |
| 위험 우선 노출 | 위험도가 높은 디스크를 우선 표시 |
| Detail Drill-down | 디스크 상세 화면에서 SMART 지표와 원인 확인 |
| 쉬운 해석 | SMART raw 값을 정상/주의/위험 등급으로 변환 |

---

## 2. 메뉴 구조

```text
사이드바
├── 대시보드
├── 디스크 목록
├── 디스크 상세
├── 진단 이력
├── 수동 진단
├── 모델 성능
└── SMART 컬럼 설명
```

---

## 3. 주요 화면

### 3-1. 대시보드

```text
[요약 카드]
전체 디스크 수 / 정상 / 주의 / 위험

[위험도 시각화]
상태별 분포 차트
최근 진단 이력 추이

[위험 디스크 카드]
위험도가 높은 디스크 우선 표시
```

### 3-2. 디스크 목록

```text
디스크 모델명
시리얼 번호
용량
최종 등급
위험도
최근 진단 시간
조치 상태
```

### 3-3. 디스크 상세

```text
[상단]
디스크 모델명 / 시리얼 / 최종 상태 / 위험도

[진단 결과]
ML 등급
규칙 기반 등급
최종 등급
권장 조치

[SMART 지표]
SMART 5, 187, 188, 194, 197, 198, 199 등 핵심 값 표시
```

### 3-4. 진단 이력

```text
디스크별 과거 진단 로그
진단 시간
위험도 변화
최종 등급 변화
```

### 3-5. 수동 진단

```text
사용자가 SMART 값을 직접 입력
Backend /api/diagnose 호출
예측 확률, 최종 등급, 위험 원인 표시
```

### 3-6. 모델 성능

모델 평가 CSV를 기반으로 주요 성능 지표를 표시한다.

| 파일 | 설명 |
|------|------|
| `storage_model_scores_all_metrics.csv` | 모델별 전체 평가 지표 |
| `storage_model_scores_threshold_optimized.csv` | threshold 최적화 기준 모델 순위 |
| `storage_threshold_search_all_models.csv` | threshold별 평가 결과 |

---

## 4. 화면 흐름

```text
대시보드
  → 위험 디스크 선택
  → 디스크 상세 확인
  → 위험 원인 및 권장 조치 확인
  → 진단 이력 확인
```

Agent 기반 자동 수집 흐름은 다음과 같다.

```text
Agent 실행
  → SMART 수집
  → Backend 전송
  → DB 저장
  → Dashboard 갱신
```
