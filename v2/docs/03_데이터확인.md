# PDFS 데이터 확인 및 모델 선택

> Backblaze SMART 데이터 기반 저장장치 고장 예측 모델 정리  
> 최종 수정: 2026-06-14

---

## 1. 데이터 출처

| 항목 | 내용 |
|------|------|
| 데이터셋 | Backblaze Hard Drive Test Data |
| 데이터 형태 | HDD SMART 로그 CSV |
| 주요 라벨 | `failure` |
| 예측 대상 | 저장장치 고장 여부 |

Backblaze 데이터는 날짜별 HDD 상태 로그를 제공하며, 각 행에는 디스크 모델명, 시리얼 번호, 용량, failure 라벨, SMART 지표가 포함된다.

---

## 2. 초기 데이터 한계

초기에는 2016년 1분기 데이터만 사용하였다. 이때 정상 데이터는 충분했지만 고장 데이터가 매우 적어 불균형 문제가 컸다.

| 구분 | 개수 |
|------|---:|
| 정상 데이터 | 45,500개 |
| 고장 데이터 | 336개 |
| 고장 비율 | 약 0.73% |

고장 예측 모델은 고장 사례를 충분히 학습해야 하므로, failure=1 데이터 부족은 모델 성능의 주요 한계였다.

---

## 3. 최종 데이터 구성

정상 데이터는 기존 2016년 1분기 기준 45,500개를 유지하고, 2016년 2분기 이후 데이터를 추가 활용하여 고장 사례를 5,000개까지 확장하였다.

| 구분 | 개수 | 비율 |
|------|---:|---:|
| 정상 데이터 (`failure=0`) | 45,500개 | 90.099% |
| 고장 데이터 (`failure=1`) | 5,000개 | 9.901% |
| 전체 | 50,500개 | 100% |

이를 통해 고장 비율을 약 0.73%에서 약 9.9%로 높여, 모델이 고장 패턴을 더 충분히 학습할 수 있도록 하였다.

## 5. 모델 비교 방식

저장장치 데이터에 대해 다음 모델들을 비교하였다.

| 모델 |
|------|
| KNN |
| DecisionTree |
| RandomForest |
| Bagging |
| GradientBoosting |
| SVM |
| LogisticRegression |

평가 시 GridSearchCV를 사용했으며, 최종 선택 기준은 고장 클래스의 **F2-score**로 설정하였다. F2-score는 Precision보다 Recall에 더 큰 가중치를 주므로, 실제 고장을 놓치면 안 되는 고장 예측 문제에 적합하다.

## 8. Threshold 기준

| 확률 범위 | ML 등급 |
|-----------|---------|
| `prob < 0.36` | 정상 |
| `0.36 ≤ prob < 0.70` | 주의 |
| `prob ≥ 0.70` | 위험 |

단, SMART 규칙 기반 판단에서 위험 지표가 감지되면 ML 등급보다 높은 등급으로 최종 보정된다.
